# 连连看小游戏

一款经典的连连看配对消除网页游戏，支持桌面和移动端浏览器。

## 🎮 游戏特色

- **经典玩法**：选择两个相同图案的格子，如果它们之间可以通过不超过2个拐点的连线连接，即可消除
- **多关卡设计**：随着关卡提升，图案种类增加，难度递增
- **智能提示**：内置提示功能，帮助找到可消除的配对
- **自动重排**：当棋盘无解时自动重新排列，确保游戏可继续
- **跨平台支持**：完美适配桌面浏览器和移动设备
- **响应式设计**：自适应不同屏幕尺寸

## 📱 运行环境

- 现代浏览器（Chrome、Firefox、Safari、Edge 等）
- 支持桌面端和移动端访问
- 无需安装，打开即用

## 🎯 游戏规则

1. **消除条件**：
   - 两个格子必须是相同的图案
   - 两个格子之间可以通过直线连接（最多2个拐点）
   - 连线不能穿过其他未消除的格子

2. **连线方式**：
   - 0个拐点：直线连接
   - 1个拐点：L型连接
   - 2个拐点：Z型或U型连接

3. **胜利条件**：消除所有图案即可过关

4. **失败条件**：倒计时结束前未完成消除

##  游戏配置

### 默认配置
```javascript
{
  rows: 10,         // 棋盘行数（纵向）
  cols: 8,          // 棋盘列数（横向）
  timeLimit: 180,   // 基础时间限制（秒），每关增加 20 秒
  hints: 3          // 提示次数
}
```

### 关卡难度
- **第1关**：10种图案，时间 180 秒
- **第2关**：12种图案，时间 200 秒
- **第3关**：14种图案，时间 220 秒
- 依此类推，每关增加 2 种图案，时间增加 20 秒
- 最多32种图案

##  功能说明

### 核心功能
- **提示按钮**：高亮显示一对可消除的图案（消耗1次提示）
- **重排按钮**：重新随机排列剩余图案
- **重开按钮**：重新开始当前关卡
- **下一关**：完成当前关卡后解锁

### UI 显示
- **关卡**：当前游戏关卡
- **得分**：累计得分（消除得分 + 时间奖励）
- **时间**：剩余时间（最后30秒红色警告）
- **进度条**：已消除图案的进度

### 核心模块

#### 1. LinkGame 类 (main.js)
游戏核心逻辑，包含：
- `generateBoard()` - 棋盘生成（确保图案成对）
- `findPath()` - BFS路径查找（最多2个拐点）
- `click()` - 点击处理和配对检测
- `eliminate()` - 图案消除和得分计算
- `checkSolvable()` - 可解性检测
- `reshuffle()` - 智能重排

#### 2. UI 渲染 (app.js)
界面和交互处理，包含：
- `renderBoard()` - 棋盘渲染
- `updateUI()` - 状态更新
- `drawPath()` - 路径绘制（SVG）
- 移动端/桌面端事件适配

### 关键技术点

1. **BFS路径查找**
   - 使用广度优先搜索算法
   - 记录拐点数量，限制最多2个
   - 支持边界外绕行

2. **图案配对保证**
   - 严格确保每种图案数量为偶数
   - 均匀分配图案数量
   - 递归验证和重试机制

3. **跨平台事件处理**
   - 移动端：使用 `touchend` 事件
   - 桌面端：使用 `click` 事件
   - 防止事件重复触发

4. **数字索引系统**
   - 内部使用 0-31 的数字索引
   - 避免 Emoji 跨平台兼容性问题
   - 显示时转换为 Emoji

## 🐛 已修复问题

1. **棋盘生成不完整**
   - 原因：图案分配算法使用取模运算导致分布不均
   - 修复：均匀分配每种图案的数量

2. **图案索引0显示为空白**
   - 原因：JavaScript 中 0 是 falsy 值
   - 修复：使用 `!== null && !== undefined` 判断

3. **移动端图案不配对**
   - 原因：同时绑定 `touchend` 和 `click` 导致重复触发
   - 修复：根据设备类型选择单一事件

4. **图案数量不成对**
   - 原因：数组长度调整时使用截断破坏配对
   - 修复：删除截断逻辑，改为递归重试
